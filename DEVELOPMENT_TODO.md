## 2. וואטסאפ — מה צריך מיניר
**מה חסר:** יניר צריך להעביר:
- מספר טלפון של החשבון העסקי (WhatsApp Business)
- גישה ל-Meta Business Suite (להוסיף את talyaosher@gmail.com כמנהל)
- או לחלופין: ליצור WhatsApp Business API דרך Meta ולהעביר את ה-Token + Phone Number ID

**אפשרות חלופית (יותר פשוטה):** להשתמש בבוט וואטסאפ רגיל (green-api / wwebjs) — יניר פשוט מחבר מספר טלפון וסורק QR.

## 3. הזמנות רכש — עדכון אחרי הוצאה
**מצב:** צריך לפתח
**אפיון:**
- כל הזמנת רכש שיוצאת נשמרת בתוך הפרויקט (תיקיית הזמנות)
- מודול חדש: "תשלומים לספקים" — מציג את כל ההזמנות מכל הפרויקטים
- עריכה מכל מקום (מהפרויקט או מהמודול) — סנכרון דו-כיווני
- אפשרות לעדכן כמויות, מחירים, סטטוס תשלום

## 4. מודול רכבים — חדש
**אפיון מוצע:**
- רשימת רכבים של החברה (מספר רכב, סוג, שנה, קילומטראז')
- שיוך רכב לעובד
- תזכורות: טסט, ביטוח, שמן, טיפול תקופתי
- דיווח דלק / הוצאות רכב
- יומן נסיעות (אופציונלי)
- התראות אוטומטיות לפני תפוגת טסט/ביטוח

## 5. שירות — התממשקות ל-FIXUS
**מצב:** בבדיקה
**FIXUS** (fixus.co.il) — אפליקציה לניהול קריאות שירות וטכנאי שטח.
**אפשרויות:**
- א. אם ל-FIXUS יש API — נמשוך קריאות שירות לתוך המערכת שלנו (לבדוק איתם)
- ב. אם אין API — ייצוא CSV/Excel מ-FIXUS וייבוא למערכת שלנו
- ג. אם רוצים לוותר על FIXUS — נבנה מודול קריאות שירות מלא בתוך המערכת

**שאלה ליניר:** האם יש לכם חשבון FIXUS בתשלום? האם הם נותנים API? או שנבנה מודול שירות מאפס?

## 6. אקסלים של רבקה
**מצב:** ממתין לקבלת הקבצים
**פעולה:** לבקש מרבקה את כל האקסלים שהיא עובדת איתם — נבנה להם ממשק מסודר במערכת

## 7. בקשות תשלום
**אפיון מוצע:**
- שליחת בקשת תשלום ממותגת (עם לוגו ארגמן) + אפשרות לצרף קובץ (חשבונית, הצעת מחיר)
- שליחה במייל / וואטסאפ
- מעקב סטטוס: נשלח / אושר / שולם
- קישור לפרויקט או לספק

## 8. חשבונות חלקיים לחיוב
**אפיון:**
- בתוך מודול פרויקטים — כפתור "הוצא חשבון חלקי"
- בוחרים אחוז / סכום מתוך הפרויקט
- יוצא PDF ממותג לאישור
- מעקב: כמה חויב, כמה נשאר, היסטוריית חשבונות

## 9. קבלני משנה
**אפיון מוצע:**
- מודול "קבלני משנה" — רשימה מרכזית של כל קבלני המשנה
- לכל קבלן: שם, טלפון, תחום, דירוג, הערות
- שיוך קבלן משנה לפרויקט ולמשימות ספציפיות
- מעקב תשלומים לקבלן משנה (מקושר למודול תשלומים)
- הזמנת עבודה לקבלן משנה (PDF ממותג)

## 10. התראות עיקבון לגבייה
**אפיון:**
- כשפרויקט עובר לסטטוס "הושלם" ויש בו תקופת עיקבון — נוצרת התראה עתידית
- העיקבון יכול להיות גם שנה-שנתיים קדימה
- התראה נשלחת בוואטסאפ + במערכת כשמגיע הזמן
- אפשרות להגדיר תקופת עיקבון בכל פרויקט

## 11. משימות קבועות בוואטסאפ
**אפיון:**
- הגדרת משימות חוזרות (יומי / שבועי / חודשי)
- שיוך לאדם ספציפי
- שליחה אוטומטית בוואטסאפ
- דוגמאות:
  - אחת לחודש: "לעבור על פרויקטים ולהעביר ישנים לארכיון"
  - אחת לחודש: "לעבור על הצעות מחיר ולסגור לא רלוונטיות"
