## איך מגיעים ל-PDF

1. נכנסים למערכת
2. תפריט → הצעות מחיר → לוחצים על הצעה קיימת
3. בדף ההצעה למעלה יש dropdown "מסחרי / פרטי" ולידו כפתור "הורד PDF"
4. לוחצים → מוריד PDF

## שני סוגי הצעות

| סוג | מה כולל ב-PDF |
|-----|---------------|
| **מסחרי** | הדר חברה + פרטי לקוח + טבלת פריטים + סיכום כספי + תנאים + פוטר |
| **פרטי** | הכל כמו מסחרי + דף אודות החברה בעמוד נפרד אחרון |

## הבעיה

ה-PDF שיוצא מהמערכת לא תקין:
- שורות טבלה נחתכות בין עמודים
- הדר (שם חברה) לא חוזר בעמוד 2+
- שוליים לא מספיקים — תוכן צמוד לקצוות
- בהצעות ארוכות (20+ שורות) העמודים לא מתחלקים נכון

## מה נוסה ולא עבד

**ניסיון 1: html2canvas + jsPDF**
צילום מסך של קומפוננטת React (QuotePreview.jsx) והדבקה כתמונה ל-PDF.
לא עבד — הלוגו מ-URL חיצוני התנפח, טקסט נחתך מצד ימין, Tailwind CSS לא רונדר נכון.

**ניסיון 2: Puppeteer + HTML string (המצב הנוכחי)**
API route ב-Vercel שמקבל HTML ומחזיר PDF. ה-HTML נבנה ב-quoteHtml.js עם table, thead, CSS page-break.
לא עבד מספיק טוב — ה-PDF עדיין נחתך בהצעות ארוכות.

---

## מה צריך שיקרה

- PDF נקי בפורמט A4, RTL, עברית
- הדר חברה + כותרות עמודות חוזרים בכל עמוד
- שום שורה לא נחתכת באמצע
- סיכום (סה"כ, מע"מ, תנאים) לא נקרע מהטבלה
- שוליים מסודרים
- בהצעה פרטית — דף אודות בעמוד נפרד בסוף

## שדות QuoteLine (כל שורה בטבלה)

```javascript
{
  quote_id: string,
  name_snapshot: string,     // שם פריט
  model_snapshot: string,    // דגם
  category_snapshot: string, // קטגוריה
  sub_category_snapshot: string,
  btu_snapshot: number,
  price_no_vat_snapshot: number, // מחיר ליחידה ללא מע"מ
  quantity: number,
  line_total: number,        // מחיר × כמות
  clause_number: string,     // מספר סעיף
  order_index: number,       // סדר בטבלה
  is_header: boolean,        // שורת כותרת (שם קטגוריה בלבד, בלי מחיר)
}
```

---

## עמודות בטבלת ההצעה

| עמודה | שדה |
|-------|-----|
| סעיף | clause_number |
| קטגוריה | category_snapshot |
| תיאור | model_snapshot / name_snapshot |
| תפוקה | btu_snapshot × quantity (BTU) |
| כמות | quantity |
| סה"כ | line_total |

---

## סיכום כספי (מתחת לטבלה)

- סכום ביניים (סה"כ שורות)
- הנחה (אם discount_percentage > 0)
- מע"מ (vat_percentage, ברירת מחדל 18%)
- סה"כ לתשלום (כולל מע"מ)

## בדיקות נדרשות

1. הצעה קצרה (3-5 שורות) — עמוד אחד
2. הצעה בינונית (10-15 שורות) — 2 עמודים, הדר חוזר
3. הצעה ארוכה (25+ שורות) — 3+ עמודים, שום שורה לא נחתכת
4. הצעה עם כותרות (is_header)
5. הצעה פרטית — דף אודות בעמוד נפרד
6. הצעה עם הנחה — שורת הנחה בסיכום
7. RTL — עברית, ₪, מספרים
8. שוליים — מרווח נקי מכל הצדדים
