# תהליך רכש — ארגמן

## מצב נוכחי

### הזרימה היום:
```
הצעת מחיר מאושרת
    ↓
כתב כמויות (שורות הצעה)
    ↓
ניהול רכש — כל שורה מציגה: כמות מאושרת, כמות הוזמנה, נותר להזמנה
    ↓
"צור הזמנה" — דיאלוג: כמות, ספק, אחראי, תאריך יעד
    ↓
נוצר PurchaseRecord (סטטוס: "יש להזמין") + משימה לאחראי
    ↓
עדכון ידני של סטטוס: יש להזמין → הוזמן → סופק חלקית → סופק מלא → שולם
    ↓
אפשרות להפיק PDF הזמנת רכש (בחירת פריטים + שם ספק)
```

### סטטוסים:
| סטטוס | משמעות |
|-------|--------|
| יש להזמין | הזמנה נוצרה, ממתינה לביצוע |
| הוזמן | ההזמנה בוצעה מול הספק |
| סופק חלקית | חלק מהסחורה הגיעה |
| סופק מלא | כל הסחורה הגיעה |
| שולם | הספק קיבל תשלום |

### 3 סוגי פריטי רכש:
1. **פריטים מכתב כמויות** — מקושרים לשורת הצעה, כפתור "צור הזמנה" פעיל
2. **פריטים ידניים** — לא מכתב כמויות, כפתור "צור הזמנה" **מושבת**
3. **גרילים** — פריטי אוורור, כפתור "צור הזמנה" **מושבת**

### מה חסר:
- [ ] הפקת הזמנת רכש ממותגת (PDF מקצועי עם לוגו ופרטי חברה)
- [ ] מעקב קבלת סחורה (quantity_delivered קיים בDB אבל לא בUI)
- [ ] העלאת חשבונית ספק
- [ ] אוטוקומפליט ספקים (היום שדה חופשי)
- [ ] הזמנה לפריטים ידניים/גרילים (כפתור מושבת)

## מה צריך לבנות?

### שאלות ללקוח:
1. **הזמנת רכש** — איך נראית היום? מה חייב להיות עליה? (לוגו, פרטי חברה, תנאי תשלום?)
2. **תהליך קבלה** — כשהסחורה מגיעה, מה עושים? צריך לסמן כמויות? להעלות תעודת משלוח?
3. **ספקים** — יש רשימת ספקים קבועה? רוצים לנהל אותם במערכת?
4. **חשבוניות ספק** — רוצים להעלות חשבונית ולקשר להזמנה?
5. **פריטים ידניים** — למה הכפתור מושבת? רוצים לאפשר הזמנה גם לפריטים שלא בכתב כמויות?
